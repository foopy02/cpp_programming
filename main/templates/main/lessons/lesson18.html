Бастапқы кезеңде OOP - бұл көп нәрсе анық емес және тым күрделі қараңғы орман. Негізінде олай емес. Біз нақты (түсініксіз) анықтамалардан алшақтап, OOP-ті қарапайым сөздермен білуді ұсынамыз. 


ҚР қарапайым сөздермен

Мысал арқылы барлығын үйрену оңайырақ болғандықтан, роботты мысал ретінде алайық, біз оны OOP жүйесіндегі сабақтарды пайдалану арқылы сипаттауға тырысамыз.


Робот жағдайында

сынып оның сызбасы болып табылады. Класс (нысан) данасы сызбаға сәйкес жасалған тұтас робот болып табылады.


Мұрагерлік робот жобасына пайдалы опцияларды қосу туралы. Мысалы, біз роботтың стандартты сызбасын аламыз және оған лазерлерді, қанаттар мен сауыттарды қосамыз. Біз негізгі функционалдығы ата-аналық сыныптан алынған мұрагер сыныпта осы әрлеу жұмыстарын орындаймыз.


Полиморфизмбарлық роботтар үшін ортақ функционалдылық болып табылады және әрбір робот бір-бірінен өте ерекшеленетіні маңызды емес. Мысалы, негізгі сыныпта біз барлық кейінгі роботтар үшін қозғалыс мүмкіндігін көрсетеміз. Әрі қарай, ізбасар класында біз робот үшін левитация мүмкіндігін, басқа сыныпта суда қозғалу мүмкіндігін және т.б. көрсете аламыз. Негізгі сызбада жазылған ортақ функционалдылық бар екені белгілі болды, бірақ оны әрбір келесі робот үшін (әр мұрагер үшін) қайта жазуға болады.


Ал инкапсуляция біз үшін роботты қорғайтын құрыш. Құрыш патчының астында сымдар мен микросұлбалар сияқты осал элементтер бар. Саңылауларды құрышпен (protected немесе private) жапқаннан кейін робот сыртқы кедергілерден толығымен қорғалған. Шындығында, біз барлық өрістерге тек әдістер арқылы қол жеткіземіз, осылайша өріске тікелей кіру жабылады.


Барлық сыныптарда әртүрлі әдістер, сондай-ақ конструкторлары бар өрістер болуы мүмкін. Әрбір класс әртүрлі нысандардың кез келген санын жасауға мүмкіндік береді, олардың барлығының өзіндік сипаттамалары бар.


Сыныптарды құру

Класс құру үшін class кілт сөзін, содан кейін сыныптың атын жазу керек. Сынып атауларын бас әріптен бастау әдеттегідей, бірақ мұны жасамасаңыз, қате пайда болмайды.


Кез келген сыныпта өрістерді (айнымалыларды), әдістерді (функцияларды), сонымен қатар конструкторларды жасауға болады.


Жаңа класс құру және оған кейбір ақпаратты қою арқылы біз оның негізінде жаңа нысандар жасай аламыз. Нысандар public модификаторымен белгіленген барлық сынып сипаттарына қол жеткізе алады.


Үш рұқсат модификаторы бар:

  • public - деректер барлық жерде, сыныпта да, оның сыртында да көрінетін болады;
  • protected - деректер тек ол жасалған сыныпта, сондай-ақ ұрпақ сыныптарында ғана көрінеді;
  • private - деректер тек жасалған сыныпта ғана көрінетін болады.

Төменде қарапайым сыныптың мысалы көрсетілген:


class Book {
public:
	int pages;
	char name;
	float weight;
 
	void getInfoBook () {
		cout << "В книге " << name << " находиться " << pages << " страниц. " << endl;
		cout << "При этом она весит " << weight << endl;
	} 
};

Осындай класс негізінде біз көптеген нысандар жасай аламыз. Бұл жағдайда әрбір нысан белгілі бір кітап болады. Әрбір нысан үшін біз бірегей деректерді көрсете аламыз: беттер саны, кітаптың атауы және оның салмағы.


Объектіні жасау үшін бізге келесі код қажет:

Book sherlock_holms; // Создание объекта
sherlock_holms.getInfoBook(); // Вызов метода класса

Бұл кодты кодтың бір жолында жазуға болады. Түсінікті болу үшін кодты екі жолға бөлеміз. 


Деректерді сыныптан нысан арқылы алу үшін нүкте қойып, біз алғымыз келетін айнымалының немесе функцияның атын көрсету керек.
