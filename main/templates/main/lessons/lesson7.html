Ерекшелік дегеніміз не?

Сіз Мәтіндік редактор бағдарламасын әзірледіңіз делік. Бағдарламада пайдаланушы жаңа файл жасай алады, оған деректерді енгізеді, содан кейін файлды жүйеде сақтай алады.


Егер код дұрыс жазылса, қателер болмайды. Бірақ пайдаланушы редакторды ашып, қажетті файлды ашты, оған деректерді жазды, содан кейін файлды компьютерден қолмен жойды, содан кейін файлды сіздің бағдарламаңыз арқылы сақтауға тырысты деп елестетіп көрейік. 


Бұл сценарийде сіз бағдарламаны бұзатын және пайдаланушының бағдарламаңызда жұмыс істеуге деген ықыласын кетіретін қатені аласыз. 


Ерекшелік бағдарламаның өзі жұмыс істеген кезде пайда болатын қате болып табылады. Шартты мәлімдемелер арқылы мұндай қателерді қадағалау әрқашан мүмкін бола бермейді, өйткені бағдарлама қазірдің өзінде жұмыс істеп тұр, біз пайдаланушының әрекетін алдын ала білмейміз, яғни қатені жасау кезінде оны «ұстап алу» керек.


Қателерді ұстау

Жоғарыда сипатталған жағдайда қателерді жасалған сәтте анықтау үшін try catch құрылымы қажет. 


Бұл құрылымның көмегімен әртүрлі пішімдер мен оқиғалардың қателерін бақылай аласыз. Сіз файлдың қате ашылуын бақылай аласыз, сыныптың қате қосылымын қадағалай аласыз, айнымалының немесе функцияның қате жазылуын қадағалай аласыз және онда не бар, тіпті сандарды нөлге бөлуді бақылай аласыз!


Try catch құрылымын іске асыру

Қатені бақылауды қосу үшін келесі кодты жазуға болады:

try { // В блоке try мы пытаемся выполнить какой-либо код
        if (variable == 0) // Проверяем делитель на ноль
            throw 277; // Запускаем исключение и передаем в него параметр
        
        cout << something / variable << endl; 
    } catch(int i) { // Здесь отлавливаем переданную ошибку
        cout << "Нам очень повезло!" << endl;
        cout << "Нам удалось избежать ошибки с индексом " << i << " - на 0 делить нельзя!";
    }

Жоғарыдағы код ерекше жағдайларды пайдаланудың өте қарапайым мысалын көрсетеді. Оның қарапайымдылығын елемеңіз, өйткені ең бастысы try catch блогының жалпы құрылысын қарастыру.


Бұл конструкцияны пайдаланған кезде, бағдарлама барысында қандай қате орын алса да, бағдарламаның өзі жұмысын тоқтатпайды және пайдаланушы оны пайдалануда жағымсыз тәжірибе алмайды.


Бірнеше қате класын қадағалау қажет болса, бірнеше ұстау блоктарын қосуға болады. Олардың барлығы шартты операторлардағы сияқты бірінің астына қосылады.
